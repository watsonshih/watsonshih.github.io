系統介紹

Introduction

QuickRecord 是一個簡單易用的活動簽到退系統，專為各類型活動設計，無需安裝任何應用程式，直接使用網頁瀏覽器即可操作。本系統透過 QR Code 掃描技術，大幅提升簽到退的效率與準確性，適合研討會、課程、會議、社團活動等場合使用。

系統分為兩個主要介面：活動籌辦者（管理員）介面和參與者介面。活動籌辦者負責設置活動、掃描 QR Code、管理參與者名單；而參與者則可以使用手機生成個人專屬的 QR Code 進行簽到退。

QuickRecord is an easy-to-use event check-in/out system designed for various types of events. It requires no application installation and operates directly through a web browser. This system uses QR code scanning technology to significantly improve the efficiency and accuracy of attendance tracking, making it ideal for seminars, courses, meetings, club activities, and more.

The system consists of two main interfaces: the Event Organizer (Admin) interface and the Participant interface. Event organizers are responsible for setting up events, scanning QR codes, and managing participant lists, while participants can use their mobile phones to generate personalized QR codes for check-in and check-out.

功能特色

Features

  • 無需伺服器：完全在瀏覽器端運行，不需架設伺服器
  • QR Code 掃描：即時掃描 QR Code 進行簽到退
  • CSV 匯入/匯出：支援預先匯入參與者名單，並可匯出完整簽到退紀錄
  • 雙語系統：支援中文和英文介面
  • 兩階段模式：支援簽到階段和簽退階段
  • 即時統計：顯示簽到/簽退人數和詳細資訊
  • 離線模式：一旦載入後，即使沒有網路連線也能正常運作
  • 隱私保護：提供選項隱藏完整姓名和 ID
  • No Server Required: Runs entirely in the browser, no server setup needed
  • QR Code Scanning: Real-time QR code scanning for check-in/out
  • CSV Import/Export: Support for pre-importing participant lists and exporting complete attendance records
  • Bilingual System: Support for both Chinese and English interfaces
  • Two-Phase Mode: Support for check-in and check-out phases
  • Real-time Statistics: Display of check-in/out counts and detailed information
  • Offline Mode: Functions normally even without an internet connection once loaded
  • Privacy Protection: Options to hide full names and IDs

系統需求

System Requirements

活動籌辦者端

  • 支援現代網頁瀏覽器（Google Chrome、Microsoft Edge、Safari、Firefox 等）
  • 具備攝影機的裝置（筆電、平板或手機）
  • 初次載入時需要網路連線

參與者端

  • 任何可瀏覽網頁的裝置
  • 初次載入時需要網路連線

注意：由於需要使用攝影機功能，建議活動籌辦者使用具備攝影機的桌機或筆電，並搭配 Chrome 或 Edge 瀏覽器以獲得最佳掃描體驗。

For Event Organizers

  • Modern web browser (Google Chrome, Microsoft Edge, Safari, Firefox, etc.)
  • Device with a camera (laptop, tablet, or smartphone)
  • Internet connection for initial loading

For Participants

  • Any device that can browse the web
  • Internet connection for initial loading

Note: Due to the need for camera functionality, it is recommended that event organizers use a desktop or laptop with a camera, paired with Chrome or Edge browsers for the best scanning experience.

設置活動

Event Setup

要開始使用 QuickRecord 進行活動簽到，請依照以下步驟設置您的活動：

  1. 訪問主頁面

    在瀏覽器中開啟 QuickRecord 網站。首頁提供兩個選項：「直接開始」或「上傳 CSV」。

  2. 選擇啟動方式

    您可以選擇：

    • 直接開始：無需預先名單，參與者可以現場登記並生成 QR Code
    • 上傳 CSV：匯入預先準備好的參與者名單（適合已有報名資料的活動）

    若您選擇直接開始，系統會帶您進入活動資訊設定頁面。

  3. 填寫活動資訊

    輸入活動名稱（必填）。此名稱將顯示在簽到頁面並用於匯出的 CSV 檔案名稱。

  4. 設定顯示選項

    調整以下顯示設定：

    • 顯示報到人數：在簽到/簽退頁面顯示人數統計
    • 顯示完整姓名：若關閉此選項，系統將隱藏部分姓名以保護隱私
    • 顯示完整 ID：若關閉此選項，系統將隱藏部分 ID 以保護隱私

  5. 開始簽到階段

    點擊「開始簽到」按鈕後，系統將啟動攝影機並進入簽到模式。

提示：如果是預先有參與者名單的活動（如研討會、課程），建議上傳 CSV 檔案，以便系統可以核對資料並提供更完整的統計。

To start using QuickRecord for event check-in, follow these steps to set up your event:

  1. Access the Main Page

    Open the QuickRecord website in your browser. The homepage provides two options: "Start Directly" or "Upload CSV".

  2. Choose a Startup Method

    You can choose:

    • Start Directly: No pre-existing list needed; participants can register on-site and generate QR codes
    • Upload CSV: Import a pre-prepared participant list (suitable for events with registration data)

    If you choose to start directly, the system will take you to the event information settings page.

  3. Enter Event Information

    Input the event name (required). This name will be displayed on the check-in page and used for the exported CSV filename.

  4. Configure Display Options

    Adjust the following display settings:

    • Show Attendance Count: Display attendance statistics on the check-in/out pages
    • Show Full Name: If disabled, the system will partially hide names to protect privacy
    • Show Full ID: If disabled, the system will partially hide IDs to protect privacy

  5. Start the Check-in Phase

    After clicking the "Start Check-in" button, the system will activate the camera and enter check-in mode.

Tip: For events with pre-registered participants (such as seminars or courses), it's recommended to upload a CSV file so that the system can verify data and provide more complete statistics.

CSV 匯入

CSV Import

QuickRecord 支援透過 CSV 檔案匯入參與者名單，讓您能夠預先載入已報名的參與者資料。以下是準備與匯入 CSV 檔案的步驟：

準備 CSV 檔案

您的 CSV 檔案必須包含以下欄位：

  • Name：參與者姓名（必填）
  • ID：參與者識別碼（必填，可以是學號、員工編號、身分證字號等）
  • Note：備註（選填，可留空）

重要：CSV 檔案的第一列必須是欄位標題，且標題必須包含「Name」和「ID」（大小寫敏感）。

CSV 範例格式

以下是標準 CSV 檔案的範例內容：

Name,ID,Note
張小明,A12345678,團隊A
李大華,B87654321,團隊B
王美麗,C13579246,特殊飲食需求

匯入步驟

  1. 下載模板（選擇性）

    如果需要範本，可以點擊主頁面的「下載 CSV 模板」按鈕下載標準格式的 CSV 檔案。

  2. 上傳 CSV 檔案

    在 QuickRecord 首頁，將您準備好的 CSV 檔案拖放到指定區域，或點擊該區域選擇檔案上傳。

  3. 檢查編碼問題

    如果上傳後發現中文字元顯示為亂碼，可以點擊「出現亂碼？」切換檔案編碼（系統支援 UTF-8 和 Big5 編碼）。

  4. 預覽參與者名單

    系統將顯示已匯入的參與者名單預覽，請確認資料正確無誤。

  5. 完成設定

    確認名單無誤後，填寫活動名稱並調整顯示設定，然後點擊「開始簽到」按鈕。

注意：請確保 CSV 檔案中的「Name」和「ID」欄位不包含逗號，以避免解析錯誤。如果資料中包含逗號，請將該欄位值用雙引號括起來。

QuickRecord supports importing participant lists via CSV files, allowing you to preload data for registered participants. Here are the steps to prepare and import a CSV file:

Preparing the CSV File

Your CSV file must include the following fields:

  • Name: Participant name (required)
  • ID: Participant identifier (required, can be a student ID, employee number, national ID, etc.)
  • Note: Additional remarks (optional, can be left empty)

Important: The first row of the CSV file must be column headers, and the headers must include "Name" and "ID" (case-sensitive).

CSV Example Format

Here is an example of a standard CSV file content:

Name,ID,Note
John Smith,A12345678,Team A
Jane Doe,B87654321,Team B
Robert Johnson,C13579246,Special dietary needs

Import Steps

  1. Download Template (Optional)

    If you need a template, you can click the "Download CSV Template" button on the main page to download a standard format CSV file.

  2. Upload CSV File

    On the QuickRecord homepage, drag and drop your prepared CSV file to the designated area, or click the area to select a file to upload.

  3. Check for Encoding Issues

    If you find that Chinese characters appear as garbled text after uploading, you can click "Encoding Issue?" to switch the file encoding (the system supports UTF-8 and Big5 encodings).

  4. Preview Participant List

    The system will display a preview of the imported participant list. Please confirm that the data is correct.

  5. Complete Setup

    After confirming the list is correct, enter the event name and adjust the display settings, then click the "Start Check-in" button.

Caution: Ensure that the "Name" and "ID" fields in your CSV file do not contain commas, to avoid parsing errors. If your data includes commas, enclose the field value in double quotes.

簽到流程

Check-in Process

設置完成後，系統將進入簽到模式。此時活動籌辦者需要使用系統掃描參與者的 QR Code 進行簽到。以下是完整的簽到流程：

活動籌辦者操作步驟

  1. 準備掃描

    進入簽到頁面後，系統會自動啟動攝影機。左側區域會顯示攝影機預覽，右側則顯示活動資訊和最後簽到資訊。

    請確保攝影機已成功啟動，掃描狀態顯示「攝影機已啟動，請對準 QR Code」。

  2. 掃描 QR Code

    請要求參與者出示其 QR Code（參與者可通過訪問 QuickRecord 的使用者頁面生成）。將攝影機對準 QR Code 直到系統成功識別並處理。

  3. 確認簽到成功

    當系統成功掃描並處理 QR Code 後，會顯示「簽到成功！」的訊息，並更新最後簽到資訊區域以顯示該參與者的資料和簽到時間。

  4. 檢視簽到名單

    頁面下方顯示兩個標籤：「所有參與者」和「已簽到參與者」。您可以切換這些標籤來檢視不同的參與者名單。

  5. 完成簽到階段

    當所有參與者完成簽到，或活動進入需要簽退的階段時，點擊右側的「結束簽到並進入簽退」按鈕。系統將詢問確認後進入簽退模式。

    注意：一旦進入簽退模式，將無法再進行簽到操作。請確保所有參與者已完成簽到再進行此操作。

特殊情況處理

  • 重複簽到：如果參與者已完成簽到，再次掃描其 QR Code 會顯示「已經簽到過了」的提示。
  • 未在名單中的參與者：如果掃描了未在預先匯入名單中的參與者 QR Code，系統會自動將其加入參與者名單，並完成簽到。
  • 攝影機無法啟動：如果攝影機無法啟動，系統會顯示錯誤訊息。請檢查瀏覽器權限和攝影機連接。

提示：為了提高掃描效率，建議在光線良好的環境下操作，並確保 QR Code 清晰可見，無反光或陰影遮擋。

After setup is complete, the system will enter check-in mode. At this point, event organizers need to use the system to scan participants' QR codes for check-in. Here is the complete check-in process:

Event Organizer Operation Steps

  1. Prepare for Scanning

    After entering the check-in page, the system will automatically activate the camera. The left area shows the camera preview, while the right side displays event information and the latest check-in information.

    Ensure that the camera has been successfully activated, with the scanning status showing "Camera is ready, please align QR Code".

  2. Scan QR Codes

    Ask participants to present their QR codes (participants can generate these by visiting the QuickRecord user page). Point the camera at the QR code until the system successfully identifies and processes it.

  3. Confirm Check-in Success

    When the system successfully scans and processes a QR code, it will display a "Check-in Success!" message and update the latest check-in information area to show that participant's data and check-in time.

  4. View Check-in List

    The bottom of the page displays two tabs: "All Participants" and "Checked-in Participants". You can switch between these tabs to view different participant lists.

  5. Complete the Check-in Phase

    When all participants have completed check-in, or when the event enters a phase requiring check-out, click the "End Check-in and Start Check-out" button on the right. The system will ask for confirmation before entering check-out mode.

    Note: Once in check-out mode, check-in operations can no longer be performed. Make sure all participants have completed check-in before performing this operation.

Handling Special Situations

  • Duplicate Check-in: If a participant has already completed check-in, scanning their QR code again will display an "Already Checked In" prompt.
  • Participants Not on the List: If you scan a QR code for a participant who is not on the pre-imported list, the system will automatically add them to the participant list and complete the check-in.
  • Camera Cannot Start: If the camera cannot start, the system will display an error message. Check browser permissions and camera connections.

Tip: To improve scanning efficiency, it is recommended to operate in a well-lit environment and ensure that the QR code is clearly visible, with no glare or shadows obscuring it.

簽退流程

Check-out Process

當活動進入需要簽退的階段時，您可以從簽到模式切換到簽退模式。簽退流程與簽到類似，但有一些重要的差異。以下是簽退操作的詳細步驟：

進入簽退模式

在簽到頁面點擊「結束簽到並進入簽退」按鈕後，系統會顯示確認對話框。確認後，系統將關閉簽到頁面的攝影機，並開啟簽退頁面。

活動籌辦者操作步驟

  1. 準備掃描

    進入簽退頁面後，系統會自動啟動攝影機。左側區域顯示攝影機預覽，右側顯示活動資訊和最後簽退資訊。

  2. 掃描 QR Code

    請要求參與者出示其 QR Code，與簽到時相同的 QR Code。將攝影機對準 QR Code 直到系統成功識別並處理。

  3. 確認簽退成功

    當系統成功掃描並處理 QR Code 後，會顯示「簽退成功！」的訊息，並更新最後簽退資訊區域以顯示該參與者的資料和簽退時間。

  4. 檢視簽退狀態

    頁面下方顯示三個標籤：「所有參與者」、「待簽退參與者」和「已簽退參與者」。您可以切換這些標籤來檢視不同狀態的參與者名單。

  5. 完成簽退階段

    當所有參與者完成簽退或活動結束時，點擊右側的「結束簽退並下載紀錄」按鈕。系統將詢問確認後下載完整的簽到退記錄 CSV 檔案，並返回主頁面。

特殊情況處理

  • 未簽到參與者：如果掃描了未完成簽到的參與者 QR Code，系統會顯示「尚未簽到，無法簽退」的提示。
  • 重複簽退：如果參與者已完成簽退，再次掃描其 QR Code 會顯示「已經簽退過了」的提示。
  • 部分參與者無需簽退：並非所有活動都需要每位參與者簽退。您可以在尚有參與者未簽退的情況下，仍點擊「結束簽退並下載紀錄」按鈕結束活動。

重要：一旦完成簽退階段並下載記錄，系統將清空所有資料並返回主頁面。請確保您已成功下載並保存簽到退記錄，因為這些資料無法在系統中恢復。

When the event enters a phase requiring check-out, you can switch from check-in mode to check-out mode. The check-out process is similar to check-in, but with some important differences. Here are the detailed steps for the check-out operation:

Entering Check-out Mode

After clicking the "End Check-in and Start Check-out" button on the check-in page, the system will display a confirmation dialog. After confirmation, the system will close the camera on the check-in page and open the check-out page.

Event Organizer Operation Steps

  1. Prepare for Scanning

    After entering the check-out page, the system will automatically activate the camera. The left area shows the camera preview, while the right side displays event information and the latest check-out information.

  2. Scan QR Codes

    Ask participants to present their QR codes, the same QR codes used for check-in. Point the camera at the QR code until the system successfully identifies and processes it.

  3. Confirm Check-out Success

    When the system successfully scans and processes a QR code, it will display a "Check-out Success!" message and update the latest check-out information area to show that participant's data and check-out time.

  4. View Check-out Status

    The bottom of the page displays three tabs: "All Participants", "Awaiting Check-out", and "Checked-out Participants". You can switch between these tabs to view participant lists in different states.

  5. Complete the Check-out Phase

    When all participants have completed check-out or the event ends, click the "End Check-out and Download Records" button on the right. The system will ask for confirmation before downloading a complete check-in/check-out record CSV file and returning to the main page.

Handling Special Situations

  • Participants Who Haven't Checked In: If you scan a QR code for a participant who has not completed check-in, the system will display a "Not checked in, cannot check out" prompt.
  • Duplicate Check-out: If a participant has already completed check-out, scanning their QR code again will display an "Already Checked Out" prompt.
  • Some Participants Don't Need to Check Out: Not all events require every participant to check out. You can still click the "End Check-out and Download Records" button to end the event even if some participants have not checked out.

Important: Once you complete the check-out phase and download the records, the system will clear all data and return to the main page. Make sure you have successfully downloaded and saved the check-in/check-out records, as this data cannot be recovered in the system.

資料匯出

Data Export

活動結束後，QuickRecord 將生成完整的簽到退記錄，並以 CSV 檔案形式下載。以下是關於資料匯出的詳細資訊：

匯出流程

  1. 在簽退頁面點擊「結束簽退並下載紀錄」按鈕
  2. 系統會顯示確認對話框
  3. 確認後，系統會自動生成並下載 CSV 檔案
  4. 下載完成後，系統會清空資料並返回主頁面

匯出 CSV 檔案格式

匯出的 CSV 檔案包含以下欄位：

  • Name：參與者姓名
  • ID：參與者識別碼
  • Note：備註資訊（如有）
  • InCSV：標示該參與者是否在原始匯入的名單中（「Yes」或「No」）
  • CheckInTime：簽到時間（格式：YYYY-MM-DD HH:MM:SS）
  • CheckOutTime：簽退時間（格式：YYYY-MM-DD HH:MM:SS），若未簽退則為空

檔案命名與儲存

匯出的 CSV 檔案將以下列格式命名：

活動名稱_Attendance_YYYYMMDD.csv

其中：

  • 活動名稱：您在設置時輸入的活動名稱
  • YYYYMMDD：當前日期（年月日）

資料處理建議

下載 CSV 檔案後，建議您：

  • 立即備份檔案至安全位置
  • 使用 Excel 或 Google 試算表等軟體開啟並檢查資料完整性
  • 如有需要，可以進一步處理資料，例如計算參與時間、篩選特定條件的參與者等

注意：匯出的 CSV 檔案使用 UTF-8 編碼。如果在 Excel 中開啟時出現中文亂碼，建議使用「資料」選單中的「從文字」功能匯入，並指定編碼為 UTF-8。

After the event ends, QuickRecord will generate complete check-in/check-out records and download them as a CSV file. Here is detailed information about data export:

Export Process

  1. Click the "End Check-out and Download Records" button on the check-out page
  2. The system will display a confirmation dialog
  3. After confirmation, the system will automatically generate and download a CSV file
  4. After the download is complete, the system will clear the data and return to the main page

Exported CSV File Format

The exported CSV file includes the following fields:

  • Name: Participant name
  • ID: Participant identifier
  • Note: Additional information (if any)
  • InCSV: Indicates whether the participant was in the originally imported list ("Yes" or "No")
  • CheckInTime: Check-in time (format: YYYY-MM-DD HH:MM:SS)
  • CheckOutTime: Check-out time (format: YYYY-MM-DD HH:MM:SS), empty if not checked out

File Naming and Storage

  • YYYYMMDD: Current date (year, month, day)

常見問題

Frequently Asked Questions

關於系統

QuickRecord 需要網路連線嗎？

QuickRecord 僅在初次載入頁面時需要網路連線。一旦載入完成，系統可以離線運作，包括掃描 QR Code、記錄簽到退資訊等。但請注意，如果重新整理頁面或關閉瀏覽器，您將需要再次連線到網路。

系統支援哪些裝置？

QuickRecord 支援任何具備現代網頁瀏覽器的裝置。對於活動籌辦者而言，需要具備攝影機的裝置（如筆記型電腦、平板或智慧型手機）。建議使用 Chrome 或 Edge 瀏覽器以獲得最佳體驗。

資料會儲存在哪裡？

所有資料僅存儲在您的瀏覽器本地記憶體中，不會上傳到任何伺服器。當您完成活動並下載 CSV 檔案後，數據將被清除。因此，請確保在結束活動前下載資料。

關於 CSV 匯入

為什麼匯入 CSV 時出現亂碼？

這通常是由於編碼問題。QuickRecord 支援 UTF-8 和 Big5 兩種編碼。如果發現中文字元顯示為亂碼，請點擊「出現亂碼？」按鈕切換編碼方式。

可以匯入超過 1000 人的名單嗎？

理論上可以，但建議將大型活動分批進行簽到，以確保瀏覽器性能穩定。或者考慮使用較高性能的設備運行系統。

關於簽到退操作

如何處理忘記簽到但需要簽退的參與者？

系統要求參與者必須先完成簽到才能進行簽退。對於錯過簽到的參與者，您可以：

  1. 回到簽到階段（必須重新設置活動）
  2. 在下載的 CSV 檔案中手動修改資料

攝影機無法啟動怎麼辦？

請檢查：

  • 瀏覽器是否獲得了攝影機使用權限
  • 是否有其他應用程式正在使用攝影機
  • 攝影機是否正常連接並運作
  • 嘗試刷新頁面或使用不同的瀏覽器

QR Code 掃描不成功怎麼辦？

提高掃描成功率的方法：

  • 確保環境光線充足
  • 保持手機螢幕亮度足夠
  • 保持攝影機與 QR Code 之間的適當距離
  • 檢查 QR Code 是否清晰可見（無反光或褶皺）

如何在不中斷簽到的情況下切換裝置？

由於資料僅存儲在本地，無法直接在裝置間轉移。建議：

  • 完成當前簽到階段，下載資料
  • 在新裝置上重新設置活動並匯入先前下載的資料
  • 繼續簽到或進入簽退階段

About the System

Does QuickRecord require an internet connection?

QuickRecord only requires an internet connection when initially loading the page. Once loaded, the system can operate offline, including scanning QR codes and recording check-in/out information. However, please note that if you refresh the page or close the browser, you will need to connect to the internet again.

What devices does the system support?

QuickRecord supports any device with a modern web browser. For event organizers, a device with a camera (such as a laptop, tablet, or smartphone) is required. It is recommended to use Chrome or Edge browsers for the best experience.

Where is the data stored?

All data is stored only in your browser's local memory and is not uploaded to any server. When you complete the event and download the CSV file, the data will be cleared. Therefore, make sure to download the data before ending the event.

About CSV Import

Why do I see garbled text when importing CSV?

This is usually due to encoding issues. QuickRecord supports two encodings: UTF-8 and Big5. If you find Chinese characters displaying as garbled text, click the "Encoding Issue?" button to switch encoding methods.

Can I import a list of more than 1000 people?

Theoretically yes, but it is recommended to divide large events into batches for check-in to ensure stable browser performance. Alternatively, consider using higher-performance equipment to run the system.

About Check-in/out Operations

How do I handle participants who forgot to check in but need to check out?

The system requires participants to complete check-in before they can check out. For participants who missed check-in, you can:

  1. Return to the check-in phase (must reset the event)
  2. Manually modify the data in the downloaded CSV file

What if the camera cannot start?

Please check:

  • Whether the browser has been granted permission to use the camera
  • Whether another application is currently using the camera
  • Whether the camera is properly connected and functioning
  • Try refreshing the page or using a different browser

What if QR code scanning is unsuccessful?

Methods to improve scanning success rate:

  • Ensure adequate environmental lighting
  • Keep the phone screen brightness sufficient
  • Maintain an appropriate distance between the camera and the QR code
  • Check that the QR code is clearly visible (no reflections or creases)

How do I switch devices without interrupting check-in?

Since data is only stored locally, it cannot be directly transferred between devices. Recommendations:

  • Complete the current check-in phase and download the data
  • Set up the event on the new device and import the previously downloaded data
  • Continue check-in or move to the check-out phase

問題排解

Troubleshooting

使用 QuickRecord 時可能會遇到一些技術問題。以下是常見問題的解決方法：

攝影機相關問題

問題：攝影機無法啟動

可能原因與解決方法：

  • 權限問題：確保瀏覽器已獲得攝影機存取權限。點擊瀏覽器地址欄的鎖頭圖示，檢查攝影機權限設定。
  • 設備佔用：關閉可能正在使用攝影機的其他應用程式或瀏覽器分頁。
  • 硬體問題：確認攝影機硬體正常運作，試著在其他應用程式中測試攝影機。
  • 瀏覽器兼容性：嘗試使用 Chrome 或 Edge 瀏覽器，它們對 WebRTC 技術支援較好。

問題：QR Code 掃描不成功或反應緩慢

可能原因與解決方法：

  • 光線不足：確保環境有充足的光線照射 QR Code。
  • 距離不適當：調整攝影機與 QR Code 之間的距離，通常 15-30 公分為佳。
  • 模糊或損壞的 QR Code：確保 QR Code 清晰完整，無破損或褶皺。
  • 螢幕亮度：調高顯示 QR Code 的設備螢幕亮度。
  • 設備效能：如果掃描反應緩慢，可能是設備效能不足，嘗試使用效能較佳的設備。

CSV 檔案問題

問題：無法上傳或解析 CSV 檔案

可能原因與解決方法：

  • 檔案格式：確保檔案是標準的 CSV 格式，以逗號分隔。
  • 欄位標題：確認第一列包含正確的欄位標題，且大小寫正確（Name, ID, Note）。
  • 特殊字元：如果資料中包含逗號，請確保將該欄位值用雙引號括起來。
  • 檔案大小：如果檔案過大，可能導致瀏覽器記憶體不足，建議分割檔案。

問題：匯出的 CSV 檔案中有亂碼

可能原因與解決方法：

  • 編碼問題：匯出的 CSV 檔案使用 UTF-8 編碼。如使用 Excel 開啟出現亂碼，請：
    1. 在 Excel 中選擇「資料」選單
    2. 選擇「從文字/CSV」或「從文字」
    3. 選擇 CSV 檔案並指定編碼為 UTF-8
    4. 完成匯入精靈

系統操作問題

問題：系統運行緩慢或當機

可能原因與解決方法：

  • 參與者數量過多：大量資料可能導致瀏覽器效能下降，建議減少一次處理的參與者數量。
  • 瀏覽器資源不足：關閉不必要的分頁和應用程式，釋放系統資源。
  • 長時間運行：如活動時間較長，可考慮定期重新啟動系統，但記得先下載當前資料。
  • 裝置過熱：長時間使用攝影機可能導致設備過熱，確保設備有適當的散熱。

問題：意外關閉或重新整理頁面導致資料遺失

可能原因與解決方法：

  • 資料恢復：不幸的是，如果頁面被重新整理或關閉，本地儲存的資料將會遺失。建議：
    1. 定期手動記錄重要資訊
    2. 在長時間活動的中間階段下載資料備份
    3. 避免在瀏覽器設定為自動更新的環境中使用系統

注意：如遇到無法解決的技術問題，建議準備備用的簽到退方案，如傳統的紙本簽到表。

You may encounter some technical issues when using QuickRecord. Here are solutions to common problems:

Camera-Related Issues

Problem: Camera Cannot Start

Possible Causes and Solutions:

  • Permission Issues: Ensure the browser has been granted access to the camera. Click the lock icon in the browser address bar to check camera permission settings.
  • Device Occupation: Close other applications or browser tabs that may be using the camera.
  • Hardware Issues: Confirm that the camera hardware is working properly; try testing the camera in other applications.
  • Browser Compatibility: Try using Chrome or Edge browsers, which have better support for WebRTC technology.

Problem: QR Code Scanning Unsuccessful or Slow

Possible Causes and Solutions:

  • Insufficient Light: Ensure there is adequate lighting on the QR code.
  • Inappropriate Distance: Adjust the distance between the camera and the QR code; typically 15-30 cm is optimal.
  • Blurry or Damaged QR Code: Ensure the QR code is clear and intact, without damage or creases.
  • Screen Brightness: Increase the screen brightness of the device displaying the QR code.
  • Device Performance: If scanning is slow, the device performance may be insufficient; try using a device with better performance.

CSV File Issues

Problem: Cannot Upload or Parse CSV File

Possible Causes and Solutions:

  • File Format: Ensure the file is in standard CSV format, separated by commas.
  • Column Headers: Confirm that the first row includes the correct field headers with proper case sensitivity (Name, ID, Note).
  • Special Characters: If the data contains commas, ensure that the field value is enclosed in double quotes.
  • File Size: If the file is too large, it may cause the browser to run out of memory; consider splitting the file.

Problem: Garbled Text in Exported CSV File

Possible Causes and Solutions:

  • Encoding Issues: Exported CSV files use UTF-8 encoding. If you see garbled text when opening in Excel:
    1. In Excel, select the "Data" menu
    2. Choose "From Text/CSV" or "From Text"
    3. Select the CSV file and specify the encoding as UTF-8
    4. Complete the import wizard

System Operation Issues

Problem: System Running Slowly or Crashing

Possible Causes and Solutions:

  • Too Many Participants: Large amounts of data may cause browser performance to decline; consider reducing the number of participants processed at once.
  • Insufficient Browser Resources: Close unnecessary tabs and applications to free up system resources.
  • Long Running Time: For longer events, consider restarting the system periodically, but remember to download the current data first.
  • Device Overheating: Using the camera for extended periods may cause the device to overheat; ensure the device has proper ventilation.

Problem: Accidental Page Close or Refresh Causing Data Loss

Possible Causes and Solutions:

  • Data Recovery: Unfortunately, if the page is refreshed or closed, locally stored data will be lost. Recommendations:
    1. Periodically manually record important information
